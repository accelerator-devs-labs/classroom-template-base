# Learning Outcomes – Programa de Onboarding (Semanas 0 a 6)

## Semana 0 – Preparación y Entorno
- [ ] Configurar correctamente VS Code, Node.js y Git.
- [ ] Clonar y trabajar con un repositorio remoto.
- [ ] Usar comandos básicos de Git (add, commit, push, pull).
- [ ] Entender la estructura básica de un proyecto.
- [ ] Conocer las reglas de comunicación y soporte con el líder.

## Semana 1 – Fundamentos de JavaScript
- [ ] Declarar variables y constantes usando let y const.
- [ ] Usar tipos de datos básicos (string, number, boolean, array, object).
- [ ] Implementar condicionales (if / else / switch).
- [ ] Utilizar ciclos (for / while) para repetir tareas.
- [ ] Definir y llamar funciones (declarativas y flecha).
- [ ] Leer y escribir en la consola del navegador.

## Semana 2 – DOM y Eventos
- [ ] Entender qué es el DOM y cómo se estructura una página.
- [ ] Seleccionar elementos usando querySelector / querySelectorAll.
- [ ] Modificar texto, clases y estilos desde JavaScript.
- [ ] Escuchar eventos como click, input y submit.
- [ ] Manejar formularios y validar datos en el frontend.
- [ ] Crear componentes simples reutilizables con JS y DOM.

## Semana 3 – APIs y Fetch
- [ ] Entender qué es una API y qué es JSON.
- [ ] Realizar peticiones HTTP usando fetch.
- [ ] Usar async / await para manejar código asíncrono.
- [ ] Manejar errores básicos en peticiones (try/catch, status).
- [ ] Renderizar datos externos en el DOM.
- [ ] Crear pequeñas aplicaciones que consumen información desde una API pública.

## Semana 4 – Backend con Node.js y Express
- [ ] Entender el rol de un servidor backend.
- [ ] Crear un proyecto básico con Node.js y Express.
- [ ] Definir rutas (GET, POST, PUT, DELETE).
- [ ] Separar lógica en controladores y rutas.
- [ ] Probar endpoints usando Postman o Thunder Client.
- [ ] Manejar respuestas y errores de forma ordenada.

## Semana 5 – Bases de Datos e Integración
- [ ] Comprender conceptos básicos de SQL (tabla, registro, columna).
- [ ] Ejecutar operaciones CRUD (CREATE, READ, UPDATE, DELETE).
- [ ] Conectar el backend a una base de datos.
- [ ] Implementar consultas desde un servicio o repositorio.
- [ ] Relacionar endpoints de la API con operaciones en la base de datos.
- [ ] Mantener un esquema simple y coherente de datos para el proyecto.

## Semana 6 – Proyecto Real en Equipo
- [ ] Trabajar con GitFlow (main, develop, feature branches).
- [ ] Colaborar en un repositorio en equipo (branch, PR, revisión). 
- [ ] Integrar frontend, backend y base de datos en un mismo proyecto.
- [ ] Documentar el proyecto (README, scripts, instrucciones).
- [ ] Preparar y presentar una demo funcional del proyecto.
- [ ] Recibir retroalimentación y proponer mejoras futuras.
